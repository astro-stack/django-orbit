Orbit turns runtime evidence into a fix plan.

Capture requests, SQL, logs, exceptions, jobs and storage events, then hand the same evidence to humans and agents so they can move from ticket or error to hypothesis, patch and verification.

How the demo works

Orbit is built to stay out of the host app's way while still giving you enough context to debug quickly and safely.

  1. 1

    Generate evidence

    Hit the demo endpoints and let Orbit record the request, query and exception trail.

  2. 2

    Read the same context

    Open `/orbit/` or connect an agent through MCP to inspect the same runtime data.

  3. 3

    Move to a fix

    Use the incident bundle or PR context to get from symptom to a concrete hypothesis fast.

Generate evidence

These endpoints produce the kinds of traces Orbit is meant to surface. Use them to verify the dashboard, the MCP server and the agent workflow.

/books/ GET

List books and generate SQL queries for the request timeline.

/books/create/ GET

Create a record and capture the insert plus the app log entry.

/slow/ GET

Simulate a slow endpoint so Orbit can surface response timing.

/log/ GET

Emit multiple log levels and verify log capture in the feed.

/error/ GET

Trigger an exception and inspect the traceback and grouping behavior.

/duplicate-queries/ GET

Produce the N+1 pattern Orbit uses to flag duplicate query work.
