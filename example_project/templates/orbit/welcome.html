Django Orbit

Orbit captures requests, SQL, logs, exceptions, jobs, cache, storage and permissions, then turns that context into MCP tools and incident bundles. Move from ticket or error to fix hypothesis, test plan and PR context without exposing more data than your agent needs.

Ticket-to-fix workflow

Use Orbit when the human symptom is vague but the app already has the runtime evidence.

1Start from the issueTicket, failing endpoint, traceback, support report or release concern.
2Attach Orbit contextMCP tools rank requests, exceptions, N+1 candidates and sensitive-field risk.
3Propose the pathGenerate hypotheses, test plan, incident bundle and PR-ready context.

Run the local loop

Generate real events, open `/orbit/`, then ask an agent to investigate with the same evidence.

# Create sample telemetry and run Django
python demo.py setup
python manage.py runserver
# Open http://localhost:8000/orbit/
# MCP: python manage.py orbit_mcp

Generate test signals

Use these endpoints to create the kinds of runtime events Orbit records.

/books/GET

List books and generate SQL query telemetry.

 /books/create/GET

Create a book and record model, SQL and log events.

 /slow/GET

Simulate a slow request for endpoint investigation.

 /log/GET

Emit logs across levels for timeline debugging.

 /error/GET

Raise an exception and capture the traceback group.

 /duplicate-queries/GET

Create duplicate queries for N+1 candidate detection.
